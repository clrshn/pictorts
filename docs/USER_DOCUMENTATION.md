# User Documentation

## 1. Purpose

This document is the starter user documentation for the **PICTO - Records Monitoring System (PICTO-RMS)**.

It is intended for office users and staff members who operate the system.

It explains the system from the user side, not the developer side.

---

## 2. Main User Modules

The main modules visible to users are:

- Dashboard
- Documents
- Financial
- To-Do
- Notifications
- Profile

Admin users may also access:

- User Management
- Office Management

---

## 3. Dashboard

The dashboard is used to quickly view:

- summary counts
- record movement
- task reminders
- recent activity
- charts and monitoring blocks

It is the main page for quick monitoring.

---

## 4. Documents Module

The Documents module allows users to:

- encode document records
- search and filter documents
- route documents
- receive documents
- open document details
- print and export reports

Important record fields may include:

- DTS number
- PICTO number
- subject/title
- originating office
- status
- remarks

---

## 5. Financial Module

The Financial module allows users to:

- encode financial records
- search and filter records
- update status and progress
- route records
- receive records
- print and export reports

Important fields may include:

- type
- description
- supplier
- PR / PO / OBR / voucher information
- status
- progress

---

## 6. To-Do Module

The To-Do module allows users to:

- create tasks
- update priorities
- set due dates
- change task status
- add subtasks
- monitor upcoming and overdue tasks

---

## 7. Notifications

The notification area informs users about:

- reminders
- updates
- approvals
- routing-related activity

Users can open the notification panel to review recent changes.

---

## 8. Reports and Printing

The system supports:

- print preview
- PDF output
- filtered reports
- selected-row reports

For best results, users should apply filters first before printing.

---

## 9. Profile

Users can manage:

- profile information
- profile photo
- notification preferences

---
